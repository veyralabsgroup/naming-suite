# Naming Patterns — DomainForge Reference

A systematic library of naming techniques with examples. Use these to generate diverse, high-quality name candidates.

---

## Pattern 1: Phonetic Construction

Build words from scratch using phonetic principles. Hard consonants (k, v, x, t, r) + short vowels = strong brand energy.

**Formula:** [hard consonant] + [short vowel] + [consonant cluster] + [vowel ending]

**Strong consonant openers:** K, V, T, R, B, Z, Dr, Str, Tr, Fl, Gr
**Strong endings:** -ra, -ta, -ix, -ex, -io, -a, -on, -ar

**Examples generated:**
- Kova (ko-va) — clean, two syllables
- Veltra — motion + structure
- Trixa — technical, memorable
- Zarka — exotic, strong
- Florix — botanical + tech
- Drova — fast, directional

## Pattern 2: Latin/Greek Root Mining

Classical roots are underused. They feel learned without being inaccessible.

**High-value Latin roots:**

| Root | Meaning | Branding angle |
|------|---------|----------------|
| lumen | light | clarity, illumination, insight |
| forma | shape, form | design, structure, output |
| nexus | connection | network, integration, link |
| arbor | tree | growth, structure, branching |
| finis | end, boundary | completion, precision |
| plex | network, woven | complexity, integration |
| vox | voice | communication, audio |
| apex | peak | excellence, achievement |
| iter | journey, path | workflow, progress |
| ratio | reason, calculation | analytics, logic |

**Derivative examples:**
- Lumora (lumen + -ora suffix) = light-like, elegant
- Formix (forma + -ix) = structured, technical
- Plexio (plex + -io) = networked platform feel
- Arbora (arbor + -a) = growth-oriented

**High-value Greek roots:**

| Root | Meaning | Branding angle |
|------|---------|----------------|
| aether | sky, upper air | cloud, elevation |
| kairos | the right moment | timing, optimization |
| phaos | light | clarity, AI, insight |
| telos | end, goal | achievement, outcomes |
| logos | word, reason | content, intelligence |
| holos | whole | complete, unified |

## Pattern 3: Semantic Blending

Take two meaningful words, truncate both, merge them. The result should flow naturally.

**Process:**
1. Pick two words with relevant meaning
2. Try 3-4 truncation points for each
3. Combine and test phonetically
4. Drop syllables until it flows

**Examples:**

| Word 1 | Word 2 | Result | Notes |
|--------|--------|--------|-------|
| flow | archive | Florc → **Florvex** | push further from obvious |
| relay | motion | Rel + Mo → **Relmo** | clean, 2 syllables |
| draft | engine | Draf + In → **Drafin** | clean |
| pulse | framework | Puls + Ra → **Pulsra** | invented, clean |
| shift | record | Shif + to → **Shifto** | better than Shifrec |

**Key insight:** The blend works when you can't tell where one word ends and the other begins.

## Pattern 4: Modified Real Words

Take a real, familiar word and make it slightly unreal. Just enough to be ownable.

**Techniques:**
- Drop the last letter: `Note → Noto`, `Frame → Frama`
- Change one vowel: `Draft → Dreft`, `Craft → Creft`
- Add one letter: `Arc → Arco`, `Flux → Fluxa`
- Swap a consonant: `Link → Linc`, `Track → Trakk`

**Strong source words for modification:**
Arc, Beam, Bolt, Cast, Core, Craft, Crest, Dash, Draft, Draw, Drift, Edge, Flow, Forge, Frame, Gleam, Grid, Helm, Hook, Jolt, Key, Lace, Lane, Layer, Lead, Lens, Link, Loop, Mark, Mesh, Mode, Node, Note, Path, Pilot, Port, Prism, Pulse, Relay, Ridge, Ring, Root, Route, Rune, Scale, Scout, Seed, Shift, Signal, Sketch, Slate, Slice, Slide, Sort, Space, Span, Sprint, Stage, Stem, Stream, Strike, Surge, Sweep, Sync, Thread, Track, Trace, Trail, Tune, Vault, View, Wave, Weave, Zone

## Pattern 5: Compound Truncation

Two complete words mashed together. Keep it under 10 chars total.

**Rule:** Both words must survive truncation and still carry meaning.

**Good compounds:**
- `byteforge` (byte + forge) — technical + creation
- `stacklane` (stack + lane) — developer + path
- `driftwork` (drift + work) — motion + structure (8 chars) ✓
- `dotforge` (dotfile + forge) — developer community signal + creation

**Strong short prefixes for compounds:**
arc, bit, bot, byte, cue, dev, dot, fly, hop, hub, ion, key, lab, low, net, node, one, ops, pop, raw, run, tap, via, web, wire, zero

## Pattern 6: The Naming Evolution Tree

Start from one root concept and branch in 5+ directions before committing.

**Template:**
```
[root concept]
 ├─ literal form (usually bad, but start here)
 ├─ metaphorical form
 ├─ phonetic variation
 ├─ Latin/Greek derivative
 └─ modified/blended form
```

**Example — root concept: "momentum"**
```
momentum
 ├─ Momentum (bad: too long, not ownable)
 ├─ Momen (truncated, passable)
 ├─ Movo (motion + -o suffix)
 ├─ Impetus (too formal, interesting)
 ├─ Kyneto (Greek kinetos = moving)
 ├─ Velo (velo = fast, Latin)
 ├─ Torque (real word, physics)
 └─ Torqa (modified, ownable) ← strong candidate
```

Always run this tree before presenting names. Ensures diversity of approach.

## Anti-Pattern Library

Names that should never be recommended.

**Banned prefixes:**
Smart, Next, Fast, Quick, Easy, Best, Top, Pro, Super, Ultra, Hyper, Mega, Power, Auto, Digital, Digi, Tech, Techno, AI (as prefix without strong reason)

**Banned suffixes:**
-Pro, -Hub, -Platform, -Solution, -Suite, -System, -Engine, -Tool, -Service, -Online, -Now, -Plus, -Max, -360, -365, -24/7

**Banned patterns:**
- Two-word generic compounds: TaskManager, FileOrganizer, DataAnalyzer
- Misspelled common words for no reason: Kool, Xpert, Gr8
- Geek-coded nonsense: l33t, h4x, pwn
- Anything ending in -ify or -ize alone: Taskify, Notionize
- Acronyms that spell nothing: BPMS, SAAS, CRMS

**The test:** Say the name out loud: *"Have you tried [name]?"* — If it sounds like you're describing a feature rather than a product, it fails.
